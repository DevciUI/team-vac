# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Что это

Статический одностраничный сайт — стилизованный «TEAM VAC OS» лендинг игры P.O.N. (Steam app 3864580). Деплоится на Cloudflare Pages (несмотря на название каталога с упоминанием GitHub Pages). Без фреймворков, без сборки, без npm: три файла грузятся браузером напрямую.

- [index.html](index.html) — разметка, секции gate / boot / denied / gallery + `<canvas>`
- [styles.css](styles.css) — оформление в ctOS-эстетике, респонсив через media queries
- [app.js](app.js) — вся логика: рендер canvas, граф нод, ввод, локализация
- [assets/pon_gifs/](assets/pon_gifs/) — пять гифок для галереи (открывается при клике на ноду PON)
- [assets/sounds/](assets/sounds/) — wav-файлы лежат, но **в коде не подключены**: все звуки синтезируются через WebAudio в функции `beep()`. Если задача про звук — учти это, не предполагай что они используются.

## Локальный запуск и деплой

Билда нет, тестов нет, линтеров нет. Локально:

```
python -m http.server 8000
```

или любой статический сервер из корня, открыть `http://localhost:8000`. Через `file://` тоже работает, но Fullscreen API и часть событий ведут себя иначе — для проверки лучше через сервер.

Деплой — Cloudflare Pages. Корень репозитория и есть корень публикации (build command не нужен, output directory — корень).

## Архитектура `app.js`

Один файл, всё в глобальной области, последовательно:

1. **DOM-кэш** ([app.js:1-14](app.js#L1-L14)) — все нужные элементы запрашиваются один раз при загрузке.
2. **Константы и состояние** ([app.js:16-48](app.js#L16-L48)) — цвета, фокусное расстояние камеры (`FOCAL`), `STEAM_URL`, флаги фаз (`bootStarted`, `bootDone`, `galleryActive`, `dragging`), параметры камеры (`yaw`, `pitch`, `zoom`, `panX`, `panY`).
3. **Локализация** — объект `copy` с тремя языками (ru/en/basic), `setLanguage()` обновляет все `[data-i18n]` ноды и подсветку кнопок.
4. **Аудио** — `beep(freq, duration, volume)` лениво создаёт `AudioContext` и играет короткий square-wave. Других звуков нет.
5. **Граф** — массив `nodes` (центральный `CORE` под индексом 0 + 13 предзаданных) и `sparks` (анимированные «трассировки»). `seedMap()` раскидывает ноды по окружности с шумом по Z и добавляет случайные связи. `findOrCreateNode(label)` ищет ноду по нормализованному тегу или создаёт новую — используется и из поиска, и при клике.
6. **Камера 3D→2D** — `rotate()` (yaw/pitch), `project()` (перспективная проекция через `FOCAL`), `worldFromScreen()` для обратного перевода. Драг ЛКМ — орбита, Shift+ЛКМ или Shift во время драга — пан, колесо — зум.
7. **Триггер галереи** — `launchActivity(text)` создаёт spark, летящий к целевой ноде; когда spark долетает в `drawSparks`, если `node.label === "PON"`, вызывается `openGallery()`.
8. **Главный цикл `loop(now)`** ([app.js:552](app.js#L552)) — `requestAnimationFrame`, считает dt, прогрессит boot, дрейфит ноды, рисует фон → HUD → связи → искры → ноды (от дальних к ближним по rz).

### Поток UI-состояний

```
startGate
  ↓ клик / Enter / Space
boot (loading bar, fullscreen request)
  ↓ через 2с в boot-таймере
denied (ACCESS DENIED, мерцает, кнопка → Steam)
  ↓ через 8с обратно в boot
boot до 9с
  ↓
сцена + queryForm (canvas интерактивен)
  ↓ клик по PON или ввод "PON"
gallery (5 тейпов, ESC закрывает)
```

Видимость секций — класс `hidden` (`display: none !important`).

### Структура ноды

`{ label, x, y, z, color, pulse, alert, links }`:
- `pulse` затухает быстро (свечение сразу после удара искрой), `alert` — медленно (красная подсветка держится дольше).
- `links` содержит индексы соседей в обе стороны; `drawLinks` дедуплицирует через ключ `min:max`.
- Красный цвет — у «опасных» нод (`PON`, `CAMERA_FEED`, `BIOMETRIC`, `BREACH`) и у всех созданных через поиск.

## Где править типичные вещи

- **Тексты UI** — объект `copy` в [app.js](app.js) (три языка). В HTML соответствующие строки помечены `data-i18n`, JS перезаписывает их при смене языка.
- **Steam-ссылка** — константа `STEAM_URL` в начале [app.js](app.js).
- **Список тейпов** — пять `<article class="tape">` в [index.html](index.html). Если меняешь количество — поправь и текст `ARCHIVE_ITEMS 05` рядом.
- **Палитра** — CSS-переменные в `:root` ([styles.css](styles.css)) и параллельный набор JS-констант в [app.js](app.js) (`BLACK`, `WHITE`, `RED`, `RED_DIM`, `MUTED`). При смене цвета меняй в обоих местах.
- **Предзаданные ноды** — массив `seeds` внутри `seedMap()`.
- **Длительность буткада** — магические числа в `loop()`: окно denied — `elapsed >= 2 && elapsed < 8`, конец boot — `elapsed >= 9`.

## Подводные камни

- Ноды адресуются по индексам в массиве (`nodes[0]` всегда `CORE`, новые добавляются в конец). Никогда не сортируй и не вырезай ноды из середины — связи в `links` сломаются.
- `assets/sounds/*.wav` существуют, но **никогда не загружаются кодом** — это остаток ассетов. Не предполагай что они играют.
- Fullscreen-запрос в `startBoot()` молча проглатывает ошибки (`.catch(() => {})`) — на `file://` или без жеста пользователя он просто не сработает, это не баг.
- `denied` мерцает через CSS-анимацию `glitchShake` (трансформ). Если правишь стили — не убирай `position: fixed`, иначе сдвиг сломает соседние секции.
- HUD-метрика `LINKS` считается через `linkCount()` делением на 2, потому что каждая связь хранится в обоих узлах.
